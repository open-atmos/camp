Wet deposition rate constants can be constant or set from an external module using the {\ttfamily \doxylink{structcamp__rxn__wet__deposition_1_1rxn__update__data__wet__deposition__t}{camp\+\_\+rxn\+\_\+wet\+\_\+deposition\+::rxn\+\_\+update\+\_\+data\+\_\+wet\+\_\+deposition\+\_\+t}} object. An {\ttfamily camp\+\_\+rxn\+\_\+wet\+\_\+deposition\+::update\+\_\+data\+\_\+wet\+\_\+deposition\+\_\+t} object should be initialized for each wet deposition reaction. These objects can then be used during solving to update the wet deposition rate from an external module.

 The key-\/value pair {\bfseries{aerosol}} {\bfseries{phase}} is required and its value must be the name of the aerosol phase undergoing wet deposition. All species within the aerosol phase in all instances of the aerosol phase will be removed according the first-\/order loss rate constant. The {\bfseries{scaling}} {\bfseries{factor}} is optional, and can be used to set a constant scaling factor for the rate constant. When a {\bfseries{scaling}} {\bfseries{factor}} is not provided, it is assumed to be 1.\+0. All other data is optional and will be available to external modules during initialization. Rate constants are in units of $s^{-1}$, and must be set using a {\ttfamily rxn\+\_\+wet\+\_\+deposition\+\_\+update\+\_\+data\+\_\+t} object.
